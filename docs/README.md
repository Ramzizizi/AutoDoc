# База правовой аналитики

**Учебный проект — Цифровая кафедра ДГТУ, Тема №5**

Веб-система для хранения, поиска и анализа правовых материалов: норм права (НПА), судебной практики и правовых заключений. Поддерживает полнотекстовый и фасетный поиск, граф связей между документами, конструктор заключений с экспортом в .docx и аналитический дашборд.

## Быстрый старт

### Через Docker (рекомендуется)

```bash
# 1. Клонировать репозиторий
git clone <repo-url>
cd legal-analytics

# 2. Создать .env из примера
cp .env.example .env
# Отредактировать .env: установить SECRET_KEY, пароль суперпользователя

# 3. Запустить
docker compose up --build

# Приложение доступно на http://localhost:8000
# Демо-пользователи создаются автоматически (см. ниже)
```

### Локально (только для разработки, SQLite)

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py load_legal_data
python manage.py runserver
```

## Функциональность

### Полнотекстовый поиск
- Единая поисковая строка по нормам, практике и заключениям
- Стемминг русского языка (`SearchVector` с конфигурацией `russian`)
- Ранжирование по релевантности (`SearchRank`)
- Подсветка совпадений в результатах (`SearchHeadline`)
- Синтаксис websearch: фразы в кавычках, оператор `-` для исключения
- GIN-индекс для быстрого поиска

### Фасетный поиск
- Фильтрация по типу записи (нормы / практика / заключения)
- Фильтрация по отрасли права со счётчиками
- Фильтрация по тегам со счётчиками
- Фильтрация по диапазону дат
- Обновление результатов через HTMX без перезагрузки страницы

### Карточки и граф связей
- Подробная карточка каждой сущности с метаданными и тегами
- Блок связанных материалов (норма → практика → заключения)
- Интерактивный граф связей на vis-network (клик по узлу открывает карточку)

### Конструктор заключения
- Корзина: добавление норм и практики из карточек или результатов поиска
- Форма: заголовок, правовой вопрос, текст заключения
- Экспорт в .docx с разделами «Нормативное обоснование» и «Судебная практика»
- RFC 5987-кодирование кириллического имени файла

### Дашборд аналитики
- Нормы по отраслям права (столбчатая диаграмма)
- Нормы по типам НПА (круговая диаграмма)
- Практика по отраслям (горизонтальная столбчатая)
- Динамика решений по годам (линейный график)
- KPI-счётчики (всего норм / практики / заключений / отраслей)

### Разграничение доступа
| Функция | Читатель | Аналитик | Редактор |
|---|:---:|:---:|:---:|
| Поиск и просмотр | ✓ | ✓ | ✓ |
| Публичные заключения | ✓ | ✓ | ✓ |
| Все заключения | — | ✓ | ✓ |
| Конструктор / дашборд | — | ✓ | ✓ |
| Редактирование в admin | — | — | ✓ |

## Тесты

```bash
pytest                    # все тесты
pytest tests/test_search.py   # только поиск
pytest -v                 # с именами тестов
```

Результат: **60 passed, 6 skipped** (6 FTS-тестов пропускаются на SQLite, запускаются на PostgreSQL).

Покрытие:
- `test_search.py` — поиск, HTMX-частичный рендер, авторизация
- `test_facets.py` — фасеты, фильтры, дата, теги
- `test_knowledge.py` — списки, карточки, граф API, is_public
- `test_builder.py` — корзина, экспорт .docx
- `test_analytics.py` — дашборд, JSON-данные для Chart.js
- `test_roles.py` — роли, декораторы, разграничение доступа
